# Design Task Progress Triggers

Read this when deciding when to call task-core progress scripts.

## Update Progress When

- A design brief or direction is accepted.
- Style references, colors, fonts, or density decisions change.
- `design-system.md` is created or materially updated.
- A component set, mockup, screenshot, or prototype is produced.
- Visual QA passes, fails, or cannot run.
- The final answer is about to claim completion.

## Evidence Examples

Good verification evidence:

- Screenshot reviewed at named desktop and mobile viewport sizes.
- Fidelity ledger says which accepted concept image or design artifact was matched.
- User explicitly approves a concept or direction.
- Accessibility or contrast check results are recorded.

Not verification evidence:

- "Looks polished."
- "Modern and clean."
- A screenshot that was not reviewed.
- A design artifact that omits tokens or component states for a multi-screen task.
